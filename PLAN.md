# Training Block – Threshold Durability (6 Weeks)
---

# CORE OBJECTIVE

Convert threshold from fragile → stable → durable

Limiter:
- Poor steady-state control
- High decoupling at threshold
- DFA a1 instability

---

# EXECUTION RULES (CRITICAL)

## All Threshold Work
- First 5 min: 90–92% FTP
- Then settle to 95%
- Final 5–10 min: allow drift to 97% ONLY if stable

## Non-negotiables
- No surging
- No power spikes > ±5W
- Smooth cadence

## Success metrics
- VI ≤ 1.05
- Decoupling ≤ 6%
- HR smooth rise

## Progression rule
Progress ONLY if:
- Decoupling ≤ 6%
- VI ≤ 1.05
- No pacing breakdown

If not:
→ Repeat the same session next week

---

# INTENSITY GUIDELINES

- Recovery: 55–60% FTP
- True recovery: 50–55%
- Z2: 65%
- Tempo: 85–90%
- Sub-threshold cap: 90%
- Threshold: 95–97%

---

# WEEKLY STRUCTURE

- Mon: Rest / Strength
- Tue: Threshold (primary session)
- Wed: Z2 + Easy run
- Thu: Controlled aerobic (tempo OR Z2)
- Fri: Easy + Run + Strength
- Sat: Group ride (counts as intensity if hard)
- Sun: Long Z2

---

# THURSDAY RULE (IMPORTANT)

If Saturday will be hard:
→ Thursday = Z2 only

If Saturday will be easy:
→ Thursday = Tempo / Sub-threshold

---

# STRENGTH

- 2x per week (Mon, Fri)
- Full body
- Low fatigue
- No failure work

---

# RUNNING

- 2x per week (Wed, Fri)
- 20–45 min Z2 only
- Must not impact bike sessions

---

# BLOCK STRUCTURE

## Week 1 (Baseline) — 27 Apr → 3 May
- Tue: 2x20 threshold
- Thu: 2x20 tempo
- Sun: 90–120 min Z2

## Week 2 (Density) — 4 May → 10 May
- Tue: 3x15 threshold
- Thu: 3x15 tempo
- Sun: 2h Z2

## Week 3 (Extension) — 11 May → 17 May
- Tue: 30 min continuous
- Thu: 1x40 @ 88–90%
- Sun: 2–2.5h Z2

## Week 4 (Deload) — 18 May → 24 May
- Tue: 2x12 threshold
- Thu: Z2 only
- Sun: 90 min Z2

## Week 5 (Durability) — 25 May → 31 May
- Tue: 2x25 threshold
- Thu: 2x20 tempo
- Sun: 2.5h Z2

## Week 6 (Stabilisation) — 1 Jun → 7 Jun
- Tue: 3x12 threshold (high quality)
- Thu: 1x30 @ 88–90%
- Sun: 2h Z2

## Test — Week of 8 Jun
- 40 min steady effort
- Start at 95%
- Build ONLY if stable

---

# KEY PRINCIPLE

This block is about control, not power.

If execution is correct:
→ FTP will rise

If execution is poor:
→ no adaptation occurs
